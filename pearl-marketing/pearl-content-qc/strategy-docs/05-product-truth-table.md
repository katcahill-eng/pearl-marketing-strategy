# Product Truth Table

Use this table to verify product claims in content.

## Core Product Claims

| Claim | Correct? | Notes |
|-------|----------|-------|
| "SCORE recommends improvements" | NO | Snapshot rating, not a recommendation engine |
| "SCORE tells you what to fix" | NO | Rates current state; Home Improvement Plan may recommend |
| "SCORE replaces inspection" | NO | Pearl is pre-inspection preparation. Inspection is diagnosis. Different functions, different moments. Conflating them is a primary adoption blocker with listing agents. |
| "SCORE diagnoses defects" | NO | Organizes uncertainty, does not diagnose. Words like "defect," "flaw," "issue," "problem" must not appear in connection with SCORE output. |
| "SCORE helps compare homes" | YES | Core use case — one of three buyer needs (Understand, Compare, Predict) |
| "SCORE helps set expectations" | YES | Core positioning — orientation and structured understanding |
| "Use SCORE in negotiations" | NO | Pearl is neutral, not negotiation leverage. "The moment Pearl is framed as a way to extract price concessions, its credibility collapses." |
| "SCORE gives buyers leverage" | NO | Pearl serves both parties equally; buyer-first means trusted, not weaponized |
| "Pearl no longer certifies homes" | NO | Pearl still certifies. Homes scoring 500+ may qualify. Scope expanded, certification capability did not go away. |
| "Pearl is a tool for sellers to market homes / get a higher price" | NO | Pearl 1.0 thinking. Pearl is buyer-first. Perceived as seller tool = loses buyer trust = loses value for everyone. |

## Accuracy and Data Claims

| Claim | Correct? | Notes |
|-------|----------|-------|
| "Unclaimed SCORE is low-confidence / inaccurate / unreliable" | NO | Model and methodology are rigorous regardless. Score reflects what is currently known. "Low confidence" is retired terminology — never use it. |
| "More data makes the SCORE more accurate" / "Claiming fixes a flawed score" | NO | More data makes the profile more complete, not the methodology more accurate. Credit score analogy: 3 accounts vs 10 accounts — both are accurate scores. "Claim your home to complete your home's story" — not "to improve accuracy." |
| "Unclaimed home SCORE offers very little value" | NO | The framework itself is the product. 79% of buyers would use it to guide inspector questions. Gaps are structured signals showing where to direct attention. |
| "A typical score means the home has problems" | NO | Most homes score typical. It means no significant performance concerns found. "Typical is a clean bill of performance health." |

## Transaction and Market Claims

| Claim | Correct? | Notes |
|-------|----------|-------|
| "A low SCORE can hurt a sale" | NO | Buyers don't choose primarily on performance. Score surfaces info that was always going to emerge — early surfacing prevents late-stage deal collapse. Pearl organizes risk already there; it doesn't introduce new risk. |
| "SCORE Report serves buy-side agent best after transaction" | NO | Most valuable before transaction hardens — before impressions form, before offer, before inspection. "Performance information delivered early is a guide. Delivered late is a disruption." |
| "Operating costs are the most important factor for buyers" | NO | Ranked last as purchase driver. Buyers fear unexpected capital events ($60K roof, failed HVAC), not $50/month utility differences. Lead with safety, comfort, confidence — not cost savings. |
| "A high SCORE guarantees a smooth transaction" | NO | Pearl establishes context, does not control outcomes. Must not be positioned as guarantee of any outcome. |
| "Pearl surfaces information sellers didn't want buyers to know" | NO | SCORE Report builds on what seller/agent published in MLS plus public records. Pearl organizes what the seller already told the market. Gaps are info seller didn't think to include, not secrets Pearl uncovered. |
| "Home performance doesn't matter to buyers" | NO | Not primary purchase driver but primary source of anxiety. 80%+ want performance info. 90% say safety influences decision. The demand exists — the tools to meet it, until Pearl, did not. |
| "Pearl needs to wait for buyer demand before agents adopt" | NO | Buyer demand already exists (80%+). Agents need normalization, not demand. Carfax embedded in dealer workflows first; consumer expectation followed. |
