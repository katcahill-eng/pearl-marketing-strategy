## Codebase Patterns
- Pearl App lives in `pearl-app/` directory with Expo SDK 54, React Native 0.81.5, TypeScript strict
- Context pattern: createContext → Provider with AsyncStorage → isLoaded gate → useCallback mutators → barrel export in index.ts → add to App.tsx chain
- AsyncStorage keys: @pearl_rebates, @pearl_improvement_paths, @pearl_symptoms, @pearl_onboarding, @pearl_is_premium, @pearl_equipment, @pearl_completed_assessments, @pearl_registry_home
- Theme: useTheme() returns { colors, spacing, typography, borderRadius, shadows, textStyles, touchTarget }
- ButtonVariant: 'primary' | 'secondary' (not 'outline')
- colors.status has: good, typical, needsWork (no 'error' key)
- Existing contexts: OnboardingContext, DeviceContext, InsightContext, NotificationContext, UtilityBillContext, EquipmentContext, SymptomContext, RebateContext, ImprovementPathContext, ScoreContext, MaintenanceContext, UserContext, RegistryContext
- Model pattern: pure TS interfaces in src/models/, barrel export from index.ts, Date fields need custom serializer
- Navigation: RootStackParamList in RootNavigator.tsx, wrapper functions per screen, callbacks via props (not useNavigation)
- Camera pattern (BillCaptureScreen): expo-image-picker, launchCameraAsync/launchImageLibraryAsync, quality 0.8, dashed-border target, preview+retake+submit states
